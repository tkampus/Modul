# Daftar Isi Modul Git

Halaman ini berisi daftar modul pembelajaran Git. Klik judul modul untuk membuka materi terkait.

---

## Urutan Belajar yang Disarankan

1. [Pengenalan Git](<Pengenalan.md>)  
   Mengenal Git, version control, repository, commit, branch, remote, dan alur kerja dasar.

2. [Instalasi dan Konfigurasi Git](<instalasi.md>)  
   Panduan instalasi Git di Windows, macOS, dan Linux, serta konfigurasi awal `user.name` dan `user.email`.

3. [Membuat Repository Git](<Membuat Repository.md>)  
   Membuat repository baru dengan `git init`, memahami folder `.git`, dan mengecek status project.

4. [Add, Status, dan Commit](<Add Status Commit.md>)  
   Mempelajari staging area, `git add`, `git status`, `git commit`, dan pesan commit yang baik.

5. [Remote, Clone, Pull, dan Push](<Remote Clone Pull Push.md>)  
   Menghubungkan repository lokal ke GitHub, melakukan clone, pull, dan push.

6. [Branching dan Checkout Switch](<Branching dan Checkout Switch.md>)  
   Membuat branch, berpindah branch, menghapus branch, dan memahami alur kerja feature branch.

7. [Merge dan Conflict](<Merge dan Conflict.md>)  
   Menggabungkan branch, memahami conflict, dan menyelesaikan conflict secara manual.

8. [Restore, Reset, dan Revert](<Restore Reset Revert.md>)  
   Mengembalikan perubahan dengan `restore`, membatalkan commit dengan `reset`, dan membuat commit pembalik dengan `revert`.

9. [Clean dan Stash](<Clean dan Stash.md>)  
   Membersihkan file yang belum dilacak menggunakan `git clean` dan menyimpan pekerjaan sementara dengan `git stash`.

10. [Log, Diff, dan Praktik Workflow](<Log Diff Workflow.md>)  
    Membaca riwayat commit, membandingkan perubahan, dan latihan workflow Git dari awal sampai akhir.

## Target Pembelajaran

Setelah menyelesaikan seluruh modul Git, peserta diharapkan mampu:

- Memahami fungsi Git sebagai version control system.
- Menginstal dan mengonfigurasi Git.
- Membuat repository lokal.
- Menyimpan perubahan menggunakan `add` dan `commit`.
- Menghubungkan project lokal ke GitHub.
- Menggunakan `pull`, `push`, dan `clone`.
- Membuat dan mengelola branch.
- Menggabungkan branch dan menyelesaikan conflict.
- Mengembalikan perubahan menggunakan `restore`, `reset`, dan `revert`.
- Membersihkan file tidak terpakai menggunakan `clean`.
- Menggunakan workflow Git sederhana dalam project nyata.
